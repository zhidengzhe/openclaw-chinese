网关与运维

介绍

网关是 OpenClaw 的核心组件，负责处理所有消息路由、会话管理和插件加载。本指南将介绍如何安装、配置和运维 OpenClaw 网关。

网关概述

OpenClaw 网关是一个长期运行的服务，它：

  • 管理所有连接到 OpenClaw 的设备（节点）
  • 路由消息到正确的技能和插件
  • 维护用户会话状态
  • 加载和管理技能
  • 提供管理界面

安装网关

作为服务安装

# 安装 OpenClaw（如果尚未安装） npm install -g openclaw@latest # 安装网关服务 openclaw onboard --install-daemon

手动运行

# 启动网关 openclaw gateway --port 18789 # 或使用默认端口 openclaw gateway

配置网关

基本配置

# 查看当前配置 openclaw config show # 编辑配置 openclaw config edit

重要配置项

  • 端口: 网关监听的端口（默认: 18789）
  • 绑定地址: 网关绑定的网络地址
  • 日志级别: 控制日志详细程度
  • 技能目录: 技能安装位置

运行与监控

启动和停止

# 启动网关服务 openclaw gateway start # 停止网关服务 openclaw gateway stop # 重启网关服务 openclaw gateway restart # 查看状态 openclaw gateway status

监控日志

# 查看实时日志 openclaw logs --follow # 查看特定技能的日志 openclaw logs --skill weather

故障排除

常见问题

网关无法启动

可能原因: 端口被占用，权限不足，配置错误

解决方案:

# 使用其他端口 openclaw gateway --port 18790 # 调试模式运行 openclaw gateway --debug # 检查日志 openclaw logs

服务无法安装

可能原因: 系统不支持，权限不足

解决方案: 手动运行网关或检查系统权限
