代理生成文本之外的所有操作都通过工具进行。

工具是代理读取文件、运行命令、浏览网页、发送消息以及与设备交互的方式。

工具、技能和插件

OpenClaw 有三个协同工作的层次：

工具是代理调用的内容

工具是代理可以调用的类型化函数（例如 exec、browser、web_search、message）。OpenClaw 附带一组内置工具，插件可以注册额外的工具。

代理将工具视为发送到模型 API 的结构化函数定义。

技能教导代理何时以及如何使用

技能是注入到系统提示中的 markdown 文件（SKILL.md）。

技能为代理提供上下文、约束和逐步指导，以有效使用工具。技能存在于您的工作空间、共享文件夹中，或随插件一起提供。

技能参考 | 创建技能

插件将所有内容打包在一起

插件是一个可以注册任何功能组合的包：渠道、模型提供商、工具、技能、语音、实时转录、实时语音、媒体理解、图像生成、视频生成、网页抓取、网页搜索等。

一些插件是核心（随 OpenClaw 一起提供），其他是外部（由社区在 npm 上发布）。

安装和配置插件 | 构建自己的插件

内置工具

这些工具随 OpenClaw 一起提供，无需安装任何插件即可使用：

工具 功能 页面
exec / process 运行 shell 命令，管理后台进程 Exec
code_execution 运行沙盒远程 Python 分析 代码执行
browser 控制 Chromium 浏览器（导航、点击、截图） 浏览器
web_search 使用 Kimi by Moonshot 搜索网页 网页搜索
web_fetch 从 URL 获取和提取可读内容 网页抓取
message 发送消息到聊天渠道 消息
read / edit / write 文件系统操作 文件操作

插件生态系统

OpenClaw 拥有丰富的插件生态系统，扩展其功能：

  • 渠道插件：连接各种聊天应用（Telegram、WhatsApp、微信等）
  • 模型提供商插件：支持不同的 AI 模型（OpenAI、Anthropic、Google 等）
  • 工具插件：添加新的工具功能
  • 技能插件：提供预定义的技能和工作流
  • 媒体插件：语音、图像、视频处理功能

提示

开始使用工具的建议：

  1. 首先熟悉内置工具，它们覆盖了大多数常见需求
  2. 根据需要安装插件扩展功能
  3. 创建自定义技能来优化特定工作流
  4. 定期检查插件更新和新功能

工具使用最佳实践

  • 安全性：谨慎使用 exec 工具，避免运行不受信任的命令
  • 效率：批量操作时使用适当的工具组合
  • 错误处理：在技能中包含错误处理和恢复步骤
  • 文档：为自定义工具和技能编写清晰的文档
  • 测试：在生产环境使用前充分测试工具配置
