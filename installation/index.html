推荐：安装脚本

最快的安装方式。它会检测您的操作系统，如果需要则安装 Node，安装 OpenClaw，并启动初始化设置。

macOS / Linux / WSL2

curl -fsSL https://openclaw.ai/install.sh | bash

Windows (PowerShell)

iwr -useb https://openclaw.ai/install.ps1 | iex

要在不运行初始化设置的情况下安装：

macOS / Linux / WSL2

curl -fsSL https://openclaw.ai/install.sh | bash -s -- --no-onboard

Windows (PowerShell)

& ([scriptblock]::Create((iwr -useb https://openclaw.ai/install.ps1))) -NoOnboard

有关所有标志和 CI/自动化选项，请参阅 安装程序内部原理。

系统要求

  • Node 24（推荐）或 Node 22.14+ — 安装脚本会自动处理此要求
  • macOS、Linux 或 Windows — 支持原生 Windows 和 WSL2；WSL2 更稳定。请参阅 Windows 平台。
  • pnpm 仅在从源代码构建时才需要

替代安装方法

本地前缀安装程序 (install-cli.sh)

当您希望将 OpenClaw 和 Node 保留在本地前缀（例如 ~/.openclaw）下，而不依赖于系统范围的 Node 安装时使用此方法：

curl -fsSL https://openclaw.ai/install-cli.sh | bash

默认支持 npm 安装，以及在同一前缀流程下的 git-checkout 安装。完整参考：安装程序内部原理。

npm、pnpm 或 bun

如果您已经自己管理 Node：

npm

npm install -g openclaw@latest openclaw onboard --install-daemon

pnpm

pnpm add -g openclaw@latest pnpm approve-builds -g openclaw onboard --install-daemon

pnpm 需要明确批准具有构建脚本的包。在第一次安装后运行 pnpm approve-builds -g。

bun

bun add -g openclaw@latest openclaw onboard --install-daemon

Bun 支持全局 CLI 安装路径。对于网关运行时，Node 仍然是推荐的守护进程运行时。

故障排除：sharp 构建错误 (npm)

如果 sharp 由于全局安装的 libvips 而失败：

SHARP_IGNORE_GLOBAL_LIBVIPS=1 npm install -g openclaw@latest

从源代码安装

适用于贡献者或希望从本地检出运行的用户：

git clone https://github.com/openclaw/openclaw.git cd openclaw pnpm install && pnpm ui:build && pnpm build pnpm link --global openclaw onboard --install-daemon

或者跳过链接，从检出目录内部使用 pnpm openclaw ...。

安装后步骤

  1. 运行初始化设置：openclaw onboard
  2. 启动网关：openclaw gateway start
  3. 验证安装：openclaw --version
  4. 开始使用：openclaw chat 或连接到消息渠道

提示

如果在安装过程中遇到问题：

  • 检查 Node 版本：node --version
  • 确保有网络连接访问 npm 注册表
  • 对于权限问题，尝试使用 sudo（Linux/macOS）或以管理员身份运行（Windows）
  • 查看 帮助页面 获取常见问题解答
