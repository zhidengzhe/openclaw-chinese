注意事项

  • 渠道可以同时运行；配置多个，OpenClaw 将按聊天路由。
  • 最快的设置通常是 Telegram（简单的机器人令牌）。WhatsApp 需要二维码配对并在磁盘上存储更多状态。
  • 群组行为因渠道而异；请参阅 群组。
  • 为安全起见，强制执行私信配对和白名单；请参阅 安全。
  • 故障排除：渠道故障排除。
  • 模型提供商单独记录；请参阅 模型提供商。

提示

选择渠道的建议：

  1. 快速开始：使用 Telegram（设置最简单）
  2. 隐私优先：使用 Signal 或 Matrix
  3. 企业环境：使用 Microsoft Teams 或 Slack
  4. 中国用户：使用微信、QQ 或飞书
  5. 通用性：使用 WhatsApp（用户最多）

设置步骤

  1. 选择渠道：根据您的需求选择合适的聊天应用
  2. 获取凭证：按照具体渠道指南获取 API 令牌或设置机器人
  3. 配置网关：在 OpenClaw 配置中添加渠道设置
  4. 测试连接：使用 openclaw channels status --probe 测试连接
  5. 开始聊天：在配置的渠道上与 OpenClaw 助手聊天

注意

大多数渠道插件已捆绑在 OpenClaw 中，无需额外安装。少数渠道（如微信）需要单独安装插件。
