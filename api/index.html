API 文档

API 概述

OpenClaw 提供完整的 REST API，允许您通过编程方式与 AI 助手交互、管理会话和配置。

基础 URL

http://localhost:18789/api/v1

API 特性

  • RESTful 设计: 标准的 REST API 设计
  • JSON 格式: 所有请求和响应都使用 JSON
  • 身份验证: 支持 JWT 和 API 密钥
  • 实时更新: 支持 WebSocket 和 Server-Sent Events
  • OpenAPI 规范: 完整的 OpenAPI 3.0 文档

提示：您可以在 http://localhost:18789/api-docs 查看交互式 API 文档。

认证

API 密钥认证

在请求头中添加 API 密钥：

Authorization: Bearer your-api-key-here

JWT 认证

获取 JWT 令牌：

POST /api/v1/auth/login Content-Type: application/json { "username": "admin", "password": "your-password" }

使用 JWT 令牌：

Authorization: Bearer eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...

生成 API 密钥

POST /api/v1/auth/api-keys Authorization: Bearer {jwt-token} Content-Type: application/json { "name": "My API Key", "expiresIn": "30d" }

消息 API

POST /api/v1/messages

发送消息给 AI 助手

请求示例

POST /api/v1/messages Authorization: Bearer {api-key} Content-Type: application/json { "content": "你好，今天天气怎么样？", "sessionId": "optional-session-id", "channel": "telegram", "metadata": { "userId": "user-123", "userName": "张三" } }

响应示例

{ "id": "msg_abc123", "content": "今天天气晴朗，气温25°C，适合外出。", "role": "assistant", "sessionId": "sess_xyz789", "timestamp": "2026-04-06T10:30:00Z", "metadata": { "model": "deepseek/deepseek-chat", "tokensUsed": 42 } }
GET /api/v1/messages

获取消息历史

查询参数

参数 类型 描述
sessionId string 过滤特定会话的消息
limit number 返回的消息数量 (默认: 50)
before string 获取此时间之前的消息 (ISO 8601)

会话 API

GET /api/v1/sessions

获取所有会话

POST /api/v1/sessions

创建新会话

创建会话请求

POST /api/v1/sessions Authorization: Bearer {api-key} Content-Type: application/json { "name": "客户支持会话", "metadata": { "customerId": "cust_123", "priority": "high" }, "agentConfig": { "model": "deepseek/deepseek-chat", "temperature": 0.7 } }
GET /api/v1/sessions/{sessionId}

获取特定会话

DELETE /api/v1/sessions/{sessionId}

删除会话
